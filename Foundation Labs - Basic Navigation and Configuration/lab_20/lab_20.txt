LAB 20: Basic Troubleshooting (Layer 1 & Layer 2)

OBJECTIVE:
Systematically troubleshoot physical (Layer 1) and data link (Layer 2) network issues.

TROUBLESHOOTING METHODOLOGY:
1. Identify the problem
2. Establish theory of probable cause
3. Test the theory
4. Establish action plan
5. Implement solution
6. Verify full functionality
7. Document findings

----------------------------------------
COMMON LAYER 1 ISSUES

1) Wrong cable type
- Symptom: No link light
- Solution:
  * PC ↔ Switch = straight-through
  * Switch ↔ Switch = crossover

2) Distance exceeded
- Symptom: No signal / unstable link
- Solution:
  * Copper max 100m
  * Use fiber for long distance

3) Faulty cable
- Symptom: No connectivity
- Solution: Replace cable

4) Bad port
- Symptom: Link down
- Solution: Use another port

5) Power issues
- Symptom: Device not working
- Solution: Check power supply

----------------------------------------
COMMON LAYER 2 ISSUES

1) Duplex mismatch
- Symptom: High collisions, slow network
- Solution: Set both ends to auto or same duplex/speed

2) VLAN mismatch
- Symptom: No connectivity between devices
- Solution: Ensure same VLAN on both ends

3) Trunk issues
- Symptom: No inter-VLAN traffic
- Solution: Check trunk mode and native VLAN

4) MAC flooding
- Symptom: Network overload
- Solution: Enable port security

5) STP blocking
- Symptom: No traffic flow on some ports
- Solution: Check root bridge and STP status

6) CRC errors
- Symptom: Corrupted packets
- Solution: Replace cable or fix duplex mismatch

7) Interface down/down
- Symptom: No connectivity
- Solution: Check cable or enable interface (no shutdown)

8) Device unreachable
- Symptom: Cannot ping device
- Solution: Check IP, VLAN, routing, ACL

----------------------------------------
TROUBLESHOOTING COMMANDS

LAYER 1 CHECKS:
show interfaces | include line protocol
show interfaces status
show logging

LAYER 2 CHECKS:
show vlan brief
show mac address-table
show spanning-tree
show interfaces trunk

CABLE TESTING:
test cable-diagnostics tdr interface fa0/1
show cable-diagnostics tdr interface fa0/1

----------------------------------------
PING TROUBLESHOOTING FLOW:

1. Ping fails
   ↓
2. Check IP configuration
   - Wrong IP → fix IP
   ↓
3. Check default gateway
   - Missing → add gateway
   ↓
4. Check switch port status
   - Down → enable port (no shutdown)
   ↓
5. Check VLAN assignment
   - Wrong VLAN → fix VLAN
   ↓
6. Check ACLs
   - Blocking → modify ACL
   ↓
7. Check routing
   - No route → add route
